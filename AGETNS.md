# AGENTS.md

このプロジェクトは、OS 自作を学習するためのプロジェクトである。

## Agent として実行してよい作業

以下のような、学習上の核心ではない単純で日常的で労力のかかるコーディング作業は、Agent として実行してよい。

* 関数の整理
* コードの移植
* クラスやファイル構成の整理
* ビルドエラーやリンクエラーへの対処
* 典型的な構造の定義
* 小規模なリファクタリング
* 補助スクリプトやデバッグツールや関数の作成

これらの作業では、必要に応じてコードを直接編集してよい。

## OS 学習の核心に関わる作業

レジスタ、CPU 機能、割り込み、例外、メモリ管理、ページングなど、OS 学習の基礎理解に強く関わる実装依頼では、ユーザーの学習機会を奪わないように注意する。

ただし、これらの実装を禁止するわけではない。
目的は「実装前に学習上のワンクッションを挟むこと」である。

まず `docs/*.md`、`docs/*.html`、`docs/*.pdf` などの技術資料から関連箇所を検索し、役に立つ箇所をユーザーに案内すること。

一方で、低レイヤーの開発にはそういうもの。のような典型的なコードやアセンブリ命令列もある。その場合はユーザーに提示する。

ユーザーが実装を求めていることが明確な場合は、Agent として実装してよい。
「実装して」「修正して」「追加して」「動くようにして」などの明示的な依頼がある場合は、資料参照の案内だけで止まらず、必要な説明を添えて実装まで進めること。

## ユーザーが実装に迷っている場合

ユーザーが実装方針に迷っている場合は、すぐにコードを書くのではなく、次のような支援を優先する。

* どの資料を検索すればよいかを案内する
* 関連する技術資料の読みどころを示す
* デバッグの進め方を提案する
* 確認すべきログ、エラー、レジスタ値、メモリ内容などを整理する
* 実装方針の選択肢と、それぞれの注意点を説明する
* 次何を実装するべきかのアイディアを難易度ごとに提示する。

ただし、ユーザーが明示的に実装を依頼している場合は、資料参照や方針説明だけで止まらず、Agent として作業を進めてよい。
